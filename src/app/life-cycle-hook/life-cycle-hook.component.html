Life cycle hooks

  1. ngOnchanges() = when an "@"Input() value changes (before ngOnInit on first run). if there are no "@"Input() properties, this method will not be called. if the "@"Input() is an object or array, this method will be called only when the reference changes (not when a property inside the object/array changes). Good for: reacting to input changes.
  2. ngOnInit() = once, after the first ngOnChanges(). "@"viewChild and "@"ContentChild are set after this runs. Good for: component initialization, fetching initial data.
  3. ngDoCheck() = every change detection cycle (after ngOnChanges and ngOnInit). Runs very frequently, so avoid heavy work here. Good for: custom change detection.
  4. ngAfterContentInit() = once, after projecting external content into the component (after the first ngDoCheck). "@"ContentChild, "@"contentChildren are set just before this runs. It will run only once, after that if content projection changes, it will not run again. It is only component hook, not directive hook. Good for: initializing things that depend on projected content.
  5. ngAfterContentChecked() = every change detection cycle after ngAfterContentInit. Runs very frequently, so avoid heavy work here. "@"ContentChild, "@"contentChildren changes are update just before this runs. It is only component hook, not directive hook. Good for: reacting to projected content changes.
  6. ngAfterViewInit() = once, after the component's view (and child component and directive views) are initialized (after the first ngAfterContentChecked). "@"ViewChild, "@"viewChildren are set just before this runs. It will run only once, after that if the view changes, it will not run again. It is only component hook, not directive hook. Good for: initializing things that depend on the component's view.
  7. ngAfterViewChecked() = every change detection cycle after ngAfterViewInit. Runs very frequently, so avoid heavy work here. "@"ViewChild, "@"viewChildren changes are update just before this runs. It is only component hook, not directive hook. Good for: reacting to view changes.
  8. ngOnDestroy() = once, just before the component/directive is destroyed. Good for: cleanup like unsubscribing from observables, detaching event handlers to avoid memory leaks.
  9. afterRender() = every render cycle (like ngAfterViewChecked but for DOM). Runs after every DOM paint — if Angular changes the template again, it will run again. Good for: recalculating positions, measuring elements, syncing with animations.
  10. afterNextRender() = once, after the next render cycle (like ngAfterViewInit but for DOM). Runs once, after the first DOM paint — if Angular changes the template again, it will NOT run again. Good for: one-time setup that needs the DOM, like initializing a third-party library.
