Angular Dependency Injection – Provider & Resolver Examples

Dependency Injection (DI) in Angular Angular has a built-in DI framework. When you ask for a service in a component (via constructor), Angular looks for a provider that tells it how to create or get that service.

Provider Types

useClass: Replace service with another class

{ provide: UserService, useClass: MockUserService }

useValue: Provide a constant value

{ provide: 'API_URL', useValue: 'https://api.example.com' }

useExisting: Alias one token to another

{ provide: LoggerService, useExisting: ConsoleLoggerService }

useFactory: Use factory function

{ provide: 'API_URL', useFactory: apiFactory, deps: [EnvironmentService] }

Resolution Modifiers

@Self() – Resolve only in current injector

@SkipSelf() – Skip current, check parent injector

@Optional() – Return null if not found

@Host() – Stop resolution at host component’s boundary

Real-time Example

Imagine you want different LoggerService in development and production: 


providers: [
  { provide: LoggerService, useClass: DevLoggerService }
]

In AppModule you can configure with useFactory to switch based on environment.
